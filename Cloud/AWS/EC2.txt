IMDS
The EC2 Instance Metadata Service provides information to the running instance through the link-local address `169.254.169.254`, including instance details, networking data, IAM role information, and user data. This service is sensitive during pentests because exposed metadata can reveal temporary credentials, startup scripts, and internal configuration that may enable privilege escalation or deeper compromise.
IMDSv2 is the more secure model because it requires session token usage, while IMDSv1 does not require a token and is more exposed to abuse in scenarios such as SSRF. AWS recommends requiring IMDSv2, and Prowler includes checks to identify instances and launch templates where IMDSv2 is not enforced.
  
  prowler aws --verbose -q --checks ec2_instance_imdsv2_enabled

SSRF
if SSRF is found in a Web Application, the following endpoints should be checked for sensitive data exposure:
  http://169.254.169.254/latest/meta-data/iam/security-credentials/ 
    grab the attached IAM role name
  http://169.254.169.254/latest/meta-data/iam/security-credentials/<rolename> 
    append the rolename discovered above to the end of the URL to obtain (usually temp) session credentials
  http://169.254.169.254/latest/user-data/
    startup scripts may reveal usernames, passwords, access tokens, application endpoints, webhooks, or more.
